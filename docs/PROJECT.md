# 🧬 Virtual Biotech CSO — Project Overview

> A **multi-agent AI system** that nominates drug targets and *shows its work* — every
> GO / NO-GO verdict is **derived deductively** from cited evidence, not generated by an LLM.

**Built for the [Multiagents Hackathon](https://multiagents-hackathon.devpost.com/) ("tokens& Hacks") · Tessl AI · London · 4 Jul 2026**

🎯 **We target the [Prometheux](https://prometheux.ai) Intelligence Prize.** Prometheux is

a **Prometheux Vadalog** program ([`prometheux_reason.py`](../skills/virtual-biotech-cso/prometheux_reason.py)):
recursive rules, `@model` / `@explain` annotations, and a *non-silenceable* structural
gap-detector that turns "a prioritization axis is missing" into a **deductive fact** that
forces the agent org to go back and do more work. The reasoning chain behind every
conclusion is replayable.

Supporting sponsor tools: **Tavily** powers live literature search; **ClawBio** is the
skill runtime each scientist division executes over.

## Why this is genuinely a multi-agent project

This is a **virtual R&D org as code** — not a single prompt chain. Distinct agent roles
plan, execute, audit, and *force re-work* on each other:

| Role | What it does |
|---|---|
| **CSO Orchestrator** | Clarifies intent, decomposes the query, routes sub-tasks, synthesizes the verdict. Runs no analysis itself. |
| **Chief of Staff** | Pre-analysis briefing — field context + data availability — so effort goes where it counts. |
| **Scientist divisions (×4)** | Target ID & Prioritization · Safety · Modality & Tractability · Clinical & Literature. Each calls real databases and records **cited edges**. |
| **Scientific Reviewer panel** | Multiple lenses (completeness · methodology · …) vote; a majority **forces a re-route** to fill missing evidence — no human in the loop. |
| **Prometheux gap-detector** | A non-silenceable panel member: a provably-missing axis is a deductive fact, so it forces re-work on its own. |

The control loop:

```
query → Chief-of-Staff briefing → CSO decomposes & routes
      → 4 scientist divisions → Scientific Reviewer panel
      → (re-route to fill gaps) → CSO synthesis → GO/NO-GO
```

The reviewer can force exactly one re-route when a prioritization axis is missing — genuine
agent autonomy acting on the evidence it just gathered.

## What it solves

Picking a drug target is the most expensive guess in biotech — **~90% of clinical programs
fail**, most because the target was wrong. We operationalise the Stanford
[*Virtual Biotech*](https://www.biorxiv.org/content/10.64898/2026.02.23.707551v1) result —
that single-cell-specific targets clear Phase I→II **40% more often** with **32% lower
adverse-event rates** — into a real, running system, and add the one thing a scientist needs
to trust an AI verdict: **provenance**.

## How it maps to the judging criteria

| Criterion | How this project answers it |
|---|---|
| **Meaningful problem** | Target selection is biotech's costliest guess; we operationalise a peer-reviewed result that predicts trial success. |
| **Technical implementation** | A true agent org — orchestrator + 4 divisions + a voting reviewer panel + a deductive gap-detector — with a re-route control loop. |
| **Effective tool use** | **Prometheux** derives the verdict; **Tavily** runs live literature search; agents call real databases (Open Targets, CELLxGENE, TCGA, DepMap, openFDA, ClinicalTrials.gov). |
| **Agent autonomy** | The reviewer panel autonomously detects missing evidence and re-routes the org to fetch it. |
| **Presentation** | One command reproduces a full verdict offline; a live mode swaps in real backends. |
